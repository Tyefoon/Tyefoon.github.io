Week 2

Define each of the following business documents with 3-5 sentences each.

RFP - An RFP is a request for proposal. This request asks vendors to submit a proposal for a product or service a company wants to purchase. The RFP should include reasons for why a company would want to use those vendor’s products or services, details about how to use them, and the terms by which the proposal will be evaluated.

MOU - An MOU is a memorandum of understanding. It is used to lay out the intentions of multiple parties to enter a binding document, like a contract. The MOU is not usually a legally binding document in of itself.

MSA - An MSA is a master service agreement. It is a contract that defines the terms of future contracts between parties. Examples can include payment terms, arbitration arrangements, etc.

SOW - An SOW is a statement of work. It is a legally binding document which describes in detail the work that must be done by each party for a particular project. An SOW is typically used as an addendum to an existing MSA.

SLA - An SLA is a service-level agreement. It lists the exact details of a service that is provided to a customer, such as contract duration, service guarantees, compensations for potential outages, how problems are to be handled, performance benchmarks, and options for termination of that service. This is a legally binding contract.

MLA - An MLA is a master license agreement. It gives third parties a license to market, sublicense, or distribute a product to consumers. It is granted by either the creator, developer, or producer of said product.

 

Week 3

https://tryhackme.com/room/introtonetworking

Write-up:

            The Introductory Networking tryhackme room is divided into nine tasks which explain various aspects of networking theory and basic networking tools. The first task is the Introduction which provides a general overview of the topics that will be covered in the following tasks. Task 2 provides an overview of the OSI Model and has eleven questions to answer at the end. Most of these questions ask which levels of the OSI Model certain activities occur. There is a research question in this task which asks, “Which layer would the FTP protocol communicate with?” The correct answer to this question is 7 (the Application Layer). Task 3 provides details on Encapsulation and has five questions which can all be answered with the information provided in the task. The fourth task explains the TCP/IP Model and has nine questions. The answers to all these questions can also be found in the provided text for that task. Tasks 5-8 are all about different networking tools and using the attack box option provided by tryhackme or a Linux OS will help answer the questions for each. Task 5 explains how to use the ping command, which is used to test if a connection to a remote resource is possible. This task has five questions which can all be answered by following the instructions of how to use the command in a Linux environment. Task 6 teaches us about the traceroute command which will map out the path in which a request takes to reach its target. This task has four questions, one of which requires information from task 4. Task 7 explains how to use the whois command which provides detailed information about domain names. I was unable to use the attack box to complete this task and instead had to use the web version of whois to answer the questions for this task. This task had seven questions, one of which required using Open-Source Intelligence (OSINT) to look up the name of a golf course that is near the registrant address for a particular domain. The last task about networking tools, task 8, gave additional information about DNS and how to utilize the dig command which allows users to manually query recursive DNS servers for information about specific domains. This task had six questions, one of which required some outside research regarding the IP addresses for Google’s public DNS servers. Task 9 is the final task provides a bit of information about potential further reading.
